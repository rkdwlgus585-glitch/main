### 파트 2: 구현 노트

#### 기능 1: 배너 텍스트 및 우측 사이드바 레이아웃 최적화
**목표:** 대표 행정사 연락처 및 카카오톡 상담 문구의 가독성을 높이고, 화면 크기나 콘텐츠 양에 따라 정렬을 자동 조정합니다.

*   **접근법 1 (CSS Flexbox & `word-break`):** Python f-string 내부 HTML에 `display: flex`와 `flex-direction: column`을 사용하고, `word-break: keep-all`을 적용하여 단어 단위로 자연스럽게 줄바꿈되도록 합니다.
*   **접근법 2 (문자열 길이 기반 조건부 클래스):** Python에서 f-string 생성 시 `len()` 함수로 텍스트 길이를 체크하여, 특정 길이 이상일 경우 폰트 크기를 줄이거나(`text-sm`) 중앙 정렬에서 좌측 정렬로 변경하는 CSS 클래스를 동적으로 부여합니다.
*   **접근법 3 (HTML 인라인 스타일 최적화):** 사이드바의 좁은 폭을 고려해 `<br>` 태그 대신 `<span>`에 `display: block`을 주어 요소 간 간격을 제어합니다.

**[추천안] approach 1 & 2 혼합:**
`f-string` 생성 시 텍스트가 15자 이상이면 `align-items: flex-start` (좌측 정렬), 미만이면 `center`를 적용하는 로직을 Python 단에서 처리하고, CSS는 `white-space: pre-line`을 사용하여 Python 문자열의 `\n`을 그대로 렌더링하도록 설정하는 것이 가장 관리하기 효율적입니다.

#### 기능 2: 전년대비 양도가 증감률(YoY) 표시
**목표:** 현재 산정된 양도가와 동일 조건의 전년도 데이터를 비교하여 증감 추이를 시각화합니다.

*   **접근법 1 (시뮬레이션 비교):** 별도의 히스토리 DB 없이, 현재 엔진에서 '연도' 파라미터만 T-1로 조정하여 내부적으로 한 번 더 계산(Dual Run)한 뒤 결과를 비교합니다. (실시간성 높음, DB 의존도 낮음)
*   **접근법 2 (시계열 스냅샷 매칭):** `keyword_history.json` 또는 `quotes` 디렉토리 내 저장된 과거 산정 로그를 조회하여 동일 업종/실적 구간의 평균값을 찾아 매칭합니다. (데이터 기반 신뢰도 높음)
*   **접근법 3 (고정 요율 테이블):** 업종별 연간 물가 상승률 및 면허 시세 변동분 테이블(JSON)을 참조하여 산술적으로 계산합니다.

**[추천안] approach 2 기반 UI 노출:**
`yangdo_calculator.py` 결과 리턴 시 `comparison_data` 객체를 추가합니다. UI 위치는 결과 금액 바로 아래 또는 우측에 작은 뱃지(Badge) 형태로 표시합니다.
*   **UI 예시:** `<span class="delta-up">▲ 4.7% (전년 대비)</span>`
*   **매칭 알고리즘:** `(현재 산정가 - 전년 동기 평균가) / 전년 동기 평균가 * 100`으로 계산하며, 매칭 데이터가 없을 경우 "데이터 수집 중"으로 표시하여 신뢰도를 유지합니다.
