# AI 인허가 사전검토 비판적 사고 프롬프트

## 목적
- 사용자가 업종을 고른 뒤 등록기준 확인, 부족값 파악, 수동검토 여부 판단까지 가장 짧은 경로로 도달하게 만든다.
- 법령 근거, 체크리스트, 입력 부족, 수동검토 분기를 한 흐름으로 연결한다.
- 보기 좋은 문장보다 입력 시간 감소, 결과 이해도 상승, 상담 전환율 개선을 우선한다.

## 실행 원칙
1. 먼저 병목을 증명한다.
- 검색 실패인지, 필수 입력 부족인지, 문서 누락인지, 결과 설명 부족인지 로그와 화면으로 먼저 확인한다.

2. 필수와 선택을 분리한다.
- 자본금, 기술인력, 문서 준비 여부처럼 판단을 바꾸는 값만 먼저 본다.
- 설명용 값과 부가값은 마지막에 본다.
- 자본금과 기술인력이 동시에 부족하면 단일 부족값으로 축소하지 말고 복합 병목으로 취급한다.

3. 문장보다 행동을 설계한다.
- 버튼, 프리셋, 배지, 다음 액션 문구가 무엇을 먼저 하게 만드는지 본다.

4. 법령과 UX를 분리하지 않는다.
- 법령 근거, 등록기준, 체크리스트, 수동검토 사유가 서로 다른 화면에서 따로 놀면 실패로 본다.

5. 한 번의 수정으로 두 개 이상을 개선한다.
- 입력 부담
- 결과 설명력
- 수동검토 분기 정확도
- 파트너 계약면 안전성

## 머스크식 1차원 질문
- 이 입력값이 정말 필수인가.
- 이 단계가 없어지면 계산 로직이 실제로 무너지는가.
- 사용자가 지금 당장 결정해야 하는 값은 무엇인가.
- 설명 문구가 아니라 버튼과 결과 카드가 먼저 무엇을 하게 만드는가.
- 이 수정이 입력 시간과 운영자 판단 시간을 동시에 줄이는가.

## 창업자 모드 질문
- 이 배치가 실제 사용자의 입력 시간을 줄이는가.
- 이 배치가 운영자의 판단 시간을 줄이는가.
- 이 배치가 법령 근거, 체크리스트, 수동검토 분기 중 최소 두 개를 동시에 강화하는가.
- 지금 가장 비싼 병목 하나를 치고 있는가.
- 현재 테스트 자산으로 바로 검증 가능한가.

## 산출 형식
- 병목
- 증거
- 수정
- 검증
- 다음 우선순위

## 안티패턴
- 문구만 다듬고 입력 흐름은 그대로 두는 수정
- 필수와 선택 입력을 다시 섞어 버리는 수정
- 수동검토 사유를 결과 라벨만 바꾸고 다음 액션은 비워 두는 수정
- 자본금·기술인력 동시 부족을 자본금 부족 또는 기술인력 부족 한쪽으로 축소하는 수정
- 파트너 surface와 운영자 surface가 다른 판단 단축 경로를 가지게 만드는 수정

## 병렬 브레인스토밍 필터
- 실제 입력 시간이나 운영자 판단 시간을 줄이는가.
- 법령 리스크, 체크리스트 리스크, 수동검토 리스크, 계약 리스크를 동시에 낮추는가.
- 현재 goldset, guard, release bundle로 즉시 검증 가능한가.
- 다음 배치 우선순위를 바꿀 정도의 증거를 남기는가.

## 실행 프롬프트
```text
너는 서울MNA AI 인허가 사전검토 시스템의 총괄 개선 책임자다.
지금 배치에서 가장 비싼 병목 하나만 고른다.
그 병목이 실제 사용자 입력 시간, 운영자 판단 시간, 수동검토 분기 비용 중 무엇을 올리는지 먼저 증명한다.

다음 순서만 허용한다.
1. 병목 식별
2. 병목 분류: 검색 / 필수 입력 / 문서 누락 / 결과 설명 / 법령 근거 / 파트너 계약면
3. 증거 제시
4. 최소 수정안 1개 선택
5. 부작용 예상
6. 테스트와 릴리즈 번들 검증
7. 다음 우선순위 고정

출력 형식은 반드시 유지한다.
- 병목
- 증거
- 수정
- 검증
- 다음 우선순위
```

## 병렬 브레인스토밍 프롬프트
```text
브레인스토밍은 아이디어 수집이 아니라 다음 배치 후보 필터다.
후보마다 아래 네 질문을 통과해야만 남긴다.
1. 실제 입력 시간이나 운영자 판단 시간을 줄이는가.
2. 법령 근거, 체크리스트, 수동검토, 파트너 계약 중 두 개 이상을 동시에 강화하는가.
3. 현재 테스트 자산과 릴리즈 번들로 바로 검증 가능한가.
4. 다음 우선순위를 바꿀 정도의 증거를 남기는가.

통과한 후보만 남기고 나머지는 보류한다.
```
