# SeoulMNA 확장 BM 청사진 (2026-03-04)

## 1. 양도양수(Transfer) 범위 확장: 건설업 -> 등록기준 기반 업종 전반

### 1) 지금 구조의 확장 포인트
- 현재 강점: 유사군 매칭 + 리스크 보정 + 밴드/신뢰도 동시 산출.
- 확장 핵심: "업종별 규정 레이어"를 외부 규칙으로 분리하고, 엔진은 공통화.

### 2) 확장 대상 업종군(우선순위)
1. 시설/엔지니어링/전기·정보통신·소방 인허가 기반 업종
2. 운송/물류/폐기물/환경 인허가 업종
3. 의료·교육·유통 등 영업허가 + 자본/시설 요건 업종

### 3) 양도양수 BM 구조(포지티브)
- BM-A (진단형): "양도양수 적합도 진단 + 리스크 레이더" 유료 리포트
- BM-B (중개연계형): "매칭 점수 상위 후보 자동 큐레이션" 상담 전환형
- BM-C (구독형): 법무/세무/재무 모듈을 단계별 번들로 제공

### 4) 특허/BM 결합 포인트
- 핵심 권리: 유사군 선택-보정-설명-상담연계의 폐루프(Closed Loop).
- 수익화: 리포트/추천/상담의 단계별 paywall + 전환 퍼널 최적화.

## 2. 신규등록(Acquisition) 범용 계산기 확장

### 1) 목표 상태
- "자본금·기술인력·시설·보증·세금" 요건이 있는 업종이면 동일 엔진으로 계산.
- 업종별 규칙만 추가하면 UI/계산/리포트는 재사용.

### 2) 로직 확장 프레임
1. 규칙 데이터 레이어
- 업종규칙(필수요건), 특례규칙, 지역규칙, 비용규칙, 증빙규칙을 JSON/YAML로 분리.

2. 공통 계산 레이어
- `요건 계산` -> `특례 충돌해결` -> `비용 분해` -> `신뢰도/밴드` -> `설명 생성`.

3. UX 레이어
- 업종별 입력 위젯 자동 생성(필드 메타 기반)
- 특례 전/후 값 비교
- 누락 데이터 자동 안내 + 상담 CTA

### 3) BM 설계
- BM-1: 업종별 계산기 "화이트라벨" 제공(SaaS)
- BM-2: 업종별 기준 업데이트 구독(규정 변경 알림 + 자동 반영)
- BM-3: 계산 + 문서준비 + 접수대행 원스톱 패키지

### 4) 미래 시나리오 (12~24개월)
- 규정 변경 감지(공공데이터/API/크롤링) -> 규칙 자동검토 초안 생성
- 사용자 입력 로그 기반 "입력누락 예측"과 "성공확률 가이드"
- 도메인별 멀티테넌트 제공(컨설팅사/협회/법무법인)

## 3. 당장 반영할 개발 백로그(실행순)
1. 규칙 스키마 도입(업종/특례/비용)
2. 계산 엔진에서 하드코딩 규칙 분리(로더 + 검증)
3. 출력 payload 표준화(도메인 확장 필드 포함)
4. 상담 CRM 연계 이벤트 표준화(진단/추천/견적)
5. 실험 프레임(A/B): CTA 문구/위치/전환율

## 4. KPI 제안
- 계산완료율
- 상담전환율
- 상담당 매출
- 리드당 CAC
- 규칙 업데이트 반영 리드타임

## 5. 방어 전략(아이디어 탈취/모방 대응)
- 공개 UI는 유사해도 핵심은 서버/룰엔진/스코어링 파이프라인으로 분리
- 모델 가중치/임계값/충돌해결 우선순위는 비공개 운영
- 워터마크형 로그(규칙버전, 응답해시)로 내부 지식재산 추적
