## 1) 코드 기반 핵심 자산 추출

### A. 양도양수 계산기 (yangdo_blackbox_api.py)
- 유사군 선택 엔진: 토큰 정규화 + fuzzy 겹침 + 연도 분포 유사도 + 다중 가중치 점수화
- 이상치 억제: 단일 토큰/복합면허 과대매칭 억제, 연도별 shape/trend/tail 패널티
- 가격 산정: 유사군 기반 앵커 + 정성/리스크 후처리 + 불확실성 할인 + 상한 가드
- 안정성 가드: 극단 부채/유동비율 로그 스케일 반영, 이익잉여금 단조 감가 보장

## 2) "핵심 2축" 설계 (양도양수 / 신규등록)

### [양도양수 축-1] 유사군 선택 및 정합성 점수화 방법
독립항 후보
- 입력 매물의 텍스트/재무/연도별 성과 벡터를 생성하고,
- 후보 매물에 대해 토큰 유사도, 포함률, 정밀도, 연도 분포 shape/trend/tail, 재무 근접도를 결합한 가중 점수를 계산하며,
- 복합면허 과대매칭 및 단일업종 outlier를 패널티 처리하여,
- 임계값 이상 후보군만 산정군으로 확정하는 방법.

종속항 후보
- bigram-jaccard 기반 fuzzy overlap 임계값(예: 0.62) 적용
- 단일 업종 검색 시 후보 토큰 과다 포함 패널티
- 3개년/5개년 구조비교를 통한 분포 패널티
- 회사형태(개인/주식/유한) 보정 가중
- 잔액 별도정산 관행 업종군(balance_excluded) 예외 규칙

UI/UX 종속항
- 사용자에게 유사군 근거(유사도/핫매치/이웃 수)를 시각적으로 제공
- 낮은 신뢰 구간에서 자동 경고/상담 유도 CTA 노출

### [양도양수 축-2] 리스크/불확실성 반영 가격 밴드 생성 방법
독립항 후보
- 유사군 기반 중심값 및 1차 범위를 산출한 후,
- 입력 신호 충족도, 극단 재무비율 안정화, 행정처분/신용등급, reorg 모드 변동성을 반영하는 후처리 계수와
- 불확실성 할인 및 상한 가드를 순차 적용해
- 최종 중심값·하한·상한 및 신뢰도를 동시 산출하는 방법.

종속항 후보
- 이익잉여금 증가 시 단조 감가를 강제하는 최종 가드
- reorg 모드별 confidence penalty 및 range boost 테이블
- missing_critical 개수 기반 범위 자동 확장
- neighbor 수 기반 보수적 확장
- post_factor 잔존 반영(retained_nudge)

UI/UX/흐름 종속항
- 위험요소별 설명 노트를 자동 생성해 결과와 함께 제공
- 추정치와 전년 대비(yoy) 지표를 함께 제시

### [신규등록 축-1] 다업종 특례 반영 요건 계산 방법
독립항 후보
- 복수 업종 입력을 수신하고 업종별 자본금/기술자 기본요건을 취득한 후,
- 동시신청 특례 및 종합·전문 혼합 특례를 반영하여 자본금 요건을 재계산하고,
- 업종간 유사직종 기술자 중복 인정과 주력분야 예외를 반영해 최종 기술자 요건을 도출하는 방법.

종속항 후보
- 자본금 특례: 1/2 한도 + 종합·전문 추가 크레딧
- 기술자 특례: 업종 family overlap 기반 경감
- 주력분야 체크박스 선택 수에 따른 기술자 기준 단계 함수
- 추가 면허 수에 비례한 행정 수임료 가산

UI/UX 종속항
- 선택 즉시 프리셋 자동 채움(auto-fill)
- 특례 적용 전/후 숫자 차이를 동시 노출
- 주력분야 선택 박스를 동적으로 생성/숨김

### [신규등록 축-2] 지역/법인상태 연동 비용 분해 및 신뢰도 산출 방법
독립항 후보
- 주소 텍스트에서 중과세 상태를 판정하고,
- 법인상태(신설/기존)에 따른 세율 배수를 적용하여 세부 비용을 계산하며,
- 필수 입력 누락/기준 체크/기술자 격차를 반영해 범위와 신뢰도를 조정하는 방법.

종속항 후보
- 중과세 배수: 설립 3배, 변경 2배
- 기업진단 법령군별 건수 multiplier
- 실지출(준비자금 제외) 별도 산출
- band 계산식: missing, unchecked, engineer_gap 가중

UI/UX/흐름 종속항
- 중간 정산표(세금/수수료/전문가 비용) 실시간 렌더링
- 상담/전화/메일 전송 CTA와 결과 payload 연동
- 초안 저장/복원(draft) 기능

## 3) 공통 플랫폼 청구항(거미줄형 확장)
- 방법항: 업종 규칙+유사군 학습형 혼합 산정 파이프라인
- 시스템항: 입력 UI, 규칙 엔진, 유사군 엔진, 후처리 엔진, 설명 생성기, 상담 연계 모듈 포함
- 매체항(CRM): 상기 방법을 수행하는 프로그램이 기록된 비일시적 컴퓨터 판독 가능 매체
- UI/UX항: 특례 전후 비교, 위험 노트 자동 생성, confidence 시각화, 클릭형 상담 전환
- 데이터 흐름항: 계산 결과를 JSON payload로 표준화하여 시트/CRM/상담 자동화에 연동

## 4) 우선 출원 핵심 (실제 "코어" Top 4)
1. 양도양수 유사군 점수화 + 이상치 억제 + 연도 shape/trend/tail 통합 (축-1)
2. 양도양수 리스크 후처리 + 불확실성 할인 + 단조 감가 가드 (축-2)
3. 신규등록 다업종 특례(자본금/기술자/주력분야) 통합 계산 (축-1)
4. 신규등록 지역중과/법인상태 연동 세부비용+confidence 동시 산출 (축-2)

## 5) 권리화 실행 제안 (실무)
- 1차: 코어 4건을 각 1개 독립항 + 10~20개 종속항으로 분리 출원
- 2차: UI/설명/상담전환 흐름을 별도 출원(디자인+BM 혼합)
- 3차: 데이터 연동(시트/CRM/자동화) 파이프라인을 분할 출원
- 방어 전략: 내부 알고리즘 임계값/가중치 표는 영업비밀로 분리 관리

## 6) 경쟁사 모방 방어를 위한 청구항 작성 포인트
- "업종 특례 규칙" 자체보다 "특례 적용 순서와 충돌해결 로직"을 권리화
- 단순 계산식이 아닌 "설명 가능한 산정흐름(노트/신뢰도/범위)"까지 묶어 청구
- 독립항은 넓게(구조/단계), 종속항은 수치 임계값과 예외 규칙으로 촘촘하게
- UI는 단순 화면이 아니라 "사용자 행동 유도+결과 신뢰 보정 루프"를 청구항에 포함
