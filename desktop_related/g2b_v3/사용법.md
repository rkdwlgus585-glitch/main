# G2B 수주역량 자동진단 사용법 (v3)

## 빠른 시작
1. `run.bat` 실행
2. 최초 1회는 `config.txt`가 열리면 `ENCODING_KEY`, `DECODING_KEY` 입력 후 저장
3. 다시 `run.bat` 실행하면 `result/`에 리포트가 생성됩니다.
4. 필요 시 환경변수 `ENCODING_KEY`, `DECODING_KEY`를 설정하면 `config.txt` 값보다 우선 사용됩니다.
5. 고객사는 `config.txt`의 `CLIENTS=` 또는 `clients.csv(name,category,crno)`로 관리할 수 있습니다.

## 자동 실행(작업 스케줄러)
1. `schedule.bat`를 관리자 권한으로 실행
2. 작업 이름 `G2B_Auto`로 매주 월요일 09:00 등록
3. 스케줄 실행은 `--non-interactive` 모드로 동작합니다.
4. `schedule_check.bat`로 `G2B_Auto`의 최근 실행 상태를 확인할 수 있습니다.

## 고급 실행 옵션
- `py collect.py --check-config --non-interactive`
  : 키/연도/고객사 설정만 검증하고 종료
- `py collect.py --year 2026 --max-pages 40 --rows 500 --refresh-profile --non-interactive`
  : 연도/페이지/행 수 오버라이드 + endpoint/profile 캐시 무시 재탐색
- `py test_api.py --json logs\api_diag.json`
  : API 진단 결과를 JSON으로 저장
- `py regression_check.py`
  : 최신 산출물 스키마 회귀 체크
- `py -m streamlit run dashboard.py`
  : SQLite 기반 운영 대시보드

## 주요 파일
- `collect.py`: 메인 수집/분석/리포트 생성 스크립트(v3)
- `run.bat`: 수동 실행 진입점
- `schedule.bat`: 작업 스케줄러 등록
- `schedule_check.bat`: 작업 스케줄러 상태 점검
- `config.txt`: 키/기준값/고객사 설정
- `clients.csv`: 고객사 CSV 관리(name,category,crno)
- `result/`: 엑셀 리포트 출력
- `result/latest.xlsx`: 최신 리포트 바로가기
- `result/latest_all.json`, `result/latest_disc.json`, `result/latest_matched.json`, `result/latest_summary.json`
  : 최신 데이터 내보내기
- `result/g2b_history.db`: 실행 이력 및 발주 데이터(SQLite)
- `logs/`: 실행 로그 출력
- `logs/g2b_api_profile_cache.json`: 성공한 endpoint/profile 조합 캐시(다음 실행 preflight 단축)
- `logs/raw/`: API 원시 응답 샘플(장애 재현용)

## 레거시 파일 안내
- `실행.bat`는 내부적으로 `run.bat`를 호출합니다.
- `자동실행등록.bat`는 내부적으로 `schedule.bat`를 호출합니다.
- `수집.py`는 구버전(v2) 스크립트로 유지되지만 기본 실행 경로에서 제외됩니다.

## 문제 해결
- 키 포함 호출에서 `HTTP 404`가 반복되면:
  - 공공데이터포털에서 해당 API 활용신청 승인 상태 확인
  - 필수 조회 파라미터(`inqryDiv`, `orderBgnYm`, `orderEndYm`) 누락 여부 확인
  - 키 재발급 후 `config.txt` 갱신 또는 환경변수 키 재설정
- 리포트 첫 시트에 `DATA_QUALITY` 배지(FULL/PARTIAL/FAIL)가 표시됩니다.
- 스케줄 실행에서 멈춤/EOF 오류가 나면:
  - `schedule.bat`로 다시 등록했는지 확인 (`--non-interactive` 적용 필요)

## Auto Follow-up Loop (New)
- `py autopilot.py`
  : Runs check-config -> API diagnostics -> collect -> regression check -> auto brainstorm report in one pass.
- `autopilot.bat`
  : Windows one-click entry point for the same loop (`--continue-on-error` preset).
- `schedule_autopilot.bat`
  : Registers weekly scheduler task `G2B_Autopilot` (MON 09:30).
- `py autopilot.py --continue-on-error`
  : Continues optional steps even if a required step fails, then still writes next actions.
- `py autopilot.py --year 2026 --max-pages 40 --rows 500`
  : Full loop with year/page overrides.
- `py autopilot.py --sim-runs 3000 --sim-seed 42 --sim-horizons 4,8,12`
  : Runs business simulation with custom Monte Carlo settings inside the autopilot loop.
- Output: `result/next_actions_latest.md` and `result/Next_Actions_YYYYmmdd_HHMMSS.md`.

## Regression Strict Mode (New)
- `py regression_check.py --strict-matched`
  : Fails when matched output is missing/empty.

## Business Ops (New)
- `py business_ops.py`
  : Generates revenue-oriented outputs from latest data.
- Main outputs:
  - `result/sales_queue_latest.csv`
  - `result/sales_accounts_latest.csv`
  - `result/client_targets_latest.csv`
  - `result/revenue_plan_latest.json`
  - `result/business_brainstorm_latest.md`
- `autopilot.py` now runs `business_ops.py` automatically unless `--skip-business-ops` is used.
- Additional output:
  - `result/sales_activity_latest.csv` (weekly outreach status tracking template)
- Additional outputs (new):
  - `result/sales_kpi_latest.json` (funnel KPI snapshot)
  - `result/sales_weekly_report_latest.md` (weekly execution report)
  - `result/sales_daily_plan_latest.md` (daily action checklist)
  - `result/sales_simulation_latest.json` (Monte Carlo pipeline simulation output)
  - `result/sales_simulation_latest.md` (simulation summary report)
- `sales_activity_latest.csv` status/owner/memo values are preserved across reruns by `business_ops.py`.
- `client_targets_latest.csv` status/memo values are preserved across reruns by `business_ops.py`.
- Simulation options:
  - `py business_ops.py --sim-runs 3000 --sim-seed 42 --sim-horizons 4,8,12`
  - `autopilot.py` reads simulation output and includes risk/upside actions in `result/next_actions_latest.md`.
