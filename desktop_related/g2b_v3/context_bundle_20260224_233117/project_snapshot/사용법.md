# G2B 수주역량 자동진단 사용법 (v3)

## 빠른 시작
1. `run.bat` 실행
2. 최초 1회는 `config.txt`가 열리면 `ENCODING_KEY`, `DECODING_KEY` 입력 후 저장
3. 다시 `run.bat` 실행하면 `result/`에 리포트가 생성됩니다.

## 자동 실행(작업 스케줄러)
1. `schedule.bat`를 관리자 권한으로 실행
2. 작업 이름 `G2B_Auto`로 매주 월요일 09:00 등록
3. 스케줄 실행은 `--non-interactive` 모드로 동작합니다.

## 주요 파일
- `collect.py`: 메인 수집/분석/리포트 생성 스크립트(v3)
- `run.bat`: 수동 실행 진입점
- `schedule.bat`: 작업 스케줄러 등록
- `config.txt`: 키/기준값/고객사 설정
- `result/`: 엑셀 리포트 출력
- `logs/`: 실행 로그 출력

## 레거시 파일 안내
- `실행.bat`는 내부적으로 `run.bat`를 호출합니다.
- `자동실행등록.bat`는 내부적으로 `schedule.bat`를 호출합니다.
- `수집.py`는 구버전(v2) 스크립트로 유지되지만 기본 실행 경로에서 제외됩니다.

## 문제 해결
- 키 포함 호출에서 `HTTP 404`가 반복되면:
  - 공공데이터포털에서 해당 API 활용신청 승인 상태 확인
  - 키 재발급 후 `config.txt` 갱신
- 스케줄 실행에서 멈춤/EOF 오류가 나면:
  - `schedule.bat`로 다시 등록했는지 확인 (`--non-interactive` 적용 필요)
