# 특허 패키지 초안 B: AI 인허가 사전검토 (2026-03-05)

## 1. 출원 목표
- 출원명(초안): `법령 기반 등록기준 자동 매핑을 이용한 건설업 인허가 사전검토 방법`
- 범위: 업종별 등록기준(자본금/기술인력/장비/기타 요건) 자동 판정 + 부족 항목 설명 + 후속 절차 가이드.
- 목표: 법령·규정 객체 데이터에 근거한 사전검토 자동화를 권리화.

## 2. 해결하려는 문제
- 기존 방식의 한계:
1. 업종별 등록요건이 분산되어 있어 확인 누락이 발생.
2. 사용자 입력 대비 부족 요건을 구조적으로 설명하지 못함.
3. 법령 개정 시 반영 이력이 불명확해 신뢰성 확보가 어려움.

## 3. 핵심 발명 포인트(요약)
1. 법령 기준 데이터 구조화:
- 업종별 등록요건을 기준 타입(자본금/기술인력/장비/기타) 단위로 객체화.
2. 입력-요건 정합 판정:
- 사용자 입력을 각 기준 타입으로 사상(mapping)하고 충족/미충족을 자동 판정.
3. 부족 항목 설명 생성:
- 미충족 항목에 대해 증빙/보완경로를 구조화해 제시.
4. 운영 추적성:
- 판정 결과와 기준 버전 메타를 함께 저장해 사후 검증 가능성 강화.

## 4. 독립항 초안(기술 관점)
1. 방법 청구항(초안)
- 건설업 인허가 사전검토를 수행하는 컴퓨터 구현 방법에 있어서,
  - 업종별 등록기준 데이터를 기준 유형별 구조로 저장하고,
  - 사용자 입력을 기준 유형별로 변환하여 대응시킨 뒤,
  - 각 기준 유형별 충족 여부를 판정하고 부족 항목을 식별하며,
  - 부족 항목별 보완정보를 포함한 사전검토 결과를 생성·출력하는 것을 특징으로 하는 방법.

2. 시스템 청구항(초안)
- 기준 데이터 저장 모듈, 입력 매핑 모듈, 기준 판정 모듈, 부족항목 설명 모듈, 결과 제공 모듈을 포함하는 시스템.

## 5. 종속항 후보
1. 기준 데이터가 법령 출처 메타(조문/시행일/버전)를 포함하는 구성.
2. 기준 유형이 자본금·기술인력·장비·기타 등록요건으로 확장 가능한 구성.
3. 미충족 항목 수 및 중요도에 따른 후속 안내 우선순위를 자동 산정하는 구성.
4. tenant별 제공범위/기능 제한이 사전검토 결과 API에 적용되는 구성.

## 6. 코드/데이터 근거(현재 구현)

## 8. 변리사 전달 체크포인트
1. 신규성 포인트:
- 단순 체크리스트가 아닌 `법령 객체화 + 입력 매핑 + 부족항목 자동 설명` 결합.
2. 진보성 포인트:
- 기준 데이터 버전 추적성과 운영형 API 적용의 결합.
3. 권리화 구조:
- 방법 + 시스템 + 기록매체 조합으로 청구항군 분리.
