[양도가산정기 초간단 사용법 - 웹사이트 탑재 기준]

0) 먼저 기억할 것
- 목표는 프로그램 실행이 아니라 웹사이트(게시판) 반영입니다.
- 고객용/운영자용 페이지는 분리 운영합니다.

1) 화면 확인(미리보기)
- 파일 탐색기에서 "양도가산정기_미리보기.bat" 더블클릭
- 브라우저가 열리면 정상입니다.
- VSCode 내부 주소(file+.vscode-resource...)는 열지 않습니다.

2) 실서버 반영(통합 실행)
- 파일 탐색기에서 "양도가산정기_웹사이트탑재_전체실행.bat" 더블클릭
- 모드 선택:
  - 1: 테스트만(게시 안 함)
  - 2: 실게시
- 2번 선택 시 고객용 WR_ID는 비워도 됩니다(자동 처리).
- 운영자용 WR_ID는 비워두면 건너뜁니다.
- 결과 파일: logs\yangdo_site_release_latest.json
- 상태 파일: logs\yangdo_site_release_state.json

3) 개별 반영(필요할 때)
- 고객용만 반영: "양도가산정기_고객게시.bat"
- 운영자용만 반영: "양도가산정기_운영자게시.bat"

4) 정상 완료 확인
- 리포트 JSON에서 "ok": true 확인
- 고객 페이지:
  - "입금가", "양도가" 단어가 없어야 정상
  - 링크가 /mna/매물번호 로 연결되면 정상
- 운영자 페이지:
  - 내부 식별자(now UID 등) 표시 확인

5) 문제 발생 시
- 우선 "양도가산정기_웹사이트탑재_전체실행.bat" 1번(테스트 모드) 재실행
- 리포트 파일(logs\yangdo_site_release_latest.json) 기준으로 원인 확인
- 실게시 리포트에서 `publish_customer_url_missing` 이 뜨면:
  - `yangdo_ai` 게시판이 없거나 권한 부족인 경우가 가장 많습니다.
  - 그누보드 관리자에서 `yangdo_ai`, `yangdo_ai_ops` 게시판 생성/권한 설정 후 재실행합니다.
